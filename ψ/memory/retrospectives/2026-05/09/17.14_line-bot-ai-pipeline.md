## Key Architecture Decisions

1. **Reply+Push pattern**: LINE reply tokens expire in 30s, AI processing can take longer. Solution: reply "⏳ กำลังประมวลผล..." immediately, then push result via push message.
2. **LLM intent detection with keyword fallback**: `detectIntent()` uses qwen3.5:cloud first, falls back to Thai keyword matching if LLM fails.
3. **Pending transaction for uncertain types**: When AI can't determine expense vs income, create transaction with `status: 'pending'` and show confirm buttons. User confirms type, then transaction is completed and balance adjusted.
4. **Fuzzy command matching**: Removed `^...$` anchors from regex patterns so "ดูยอ" matches "ดูยอด", handles typos naturally.
5. **Single working directory**: Deleted duplicate root folder `/Users/doctorboyz/myfam` to prevent confusion.

## Honest Feedback

1. **Secret management was error-prone**: Manually copying secrets between .env files led to a truncated LINE_CHANNEL_SECRET that caused 403 errors. Should have used `diff` to verify or scripted the transfer. This wasted ~20 minutes of debugging.

2. **Scope creep on category selection**: I started implementing `buildCategoryGroupReply` and `buildSubcategoryReply` in ollama.ts and added handlers in the webhook, but the full flow (user taps group → sees subcategories → selects one) isn't end-to-end tested. I should have either committed to completing it or explicitly scoped it out.

3. **No automated testing for LINE integration**: Everything was tested manually through the LINE app or sandbox curl. There are no tests for signature verification, intent detection, or the reply+push pattern. This is a significant gap for a production feature.

## Lessons Learned

1. **Always verify secrets byte-for-byte when transferring between environments** — a single character truncation causes silent failures that look like auth errors
2. **LINE webhook verification requires a GET handler** — LINE's "Verify" button sends a GET request first, then a POST with events. Without a GET handler returning 200, verification fails.
3. **Reply token expires in 30 seconds** — for any AI processing that might take longer, always use the reply+push pattern: reply immediately with "processing", then push the result separately
4. **Fuzzy command matching improves UX significantly** — Thai users will type "ดูยอ" instead of "ดูยอด", and the bot should handle that gracefully
