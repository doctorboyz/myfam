## Key Code Changes

### Invite Link Format
```typescript
const getInviteLink = () => {
    const liffId = process.env.NEXT_PUBLIC_LIFF_ID;
    if (liffId) {
      return `https://liff.line.me/${liffId}/link?code=${inviteState!.code}`;
    }
    const baseUrl = process.env.NEXT_PUBLIC_LIFF_URL || window.location.origin;
    return `${baseUrl}/link?code=${inviteState!.code}`;
};
```

### LIFF Auto-Zoom Prevention
```css
body {
  text-size-adjust: 100%;
  -webkit-text-size-adjust: 100%;
}
```

## Architecture Decisions

- **Uniform font sizes over dynamic shrinking**: Instead of shrinking based on character count (which had edge cases), we went with a flat 9px for all secondary labels. Simpler, predictable, and looks fine in LIFF.
- **LIFF URL for invites**: Using `liff.line.me/{id}` ensures the invitee stays inside LINE's ecosystem. The fallback to origin is only for non-LIFF environments.

## Honest Feedback (100+ words, 3 friction points)

**Friction 1 — Dynamic font sizing was the wrong call.** I introduced character-count-based font shrinking because it seemed clever. The user didn't ask for it. They asked for "smaller fonts if overflow." A flat 9px would have been simpler and faster. I rationalized the complexity as "more robust" but it was just more work for marginal gain.

**Friction 2 — Incomplete cleanup.** After switching to uniform sizes, I left the old infrastructure in place: `getLabelClass` stub, `labelSmall` class, `maxLabelLen` logic in BottomNav. These are small but they confuse future readers. I should have done a pass to strip unused code.

**Friction 3 — LIFF URL format should have been there from the start.** The invite feature was built with `window.location.origin` as the link base. That's wrong for a LIFF app. I had the LIFF ID in env vars. I should have used `liff.line.me/{id}` from the first commit. The user caught it, not me.

## Lessons Learned

1. **Ask before adding dynamism.** Flat values are often what users actually want.
2. **Clean up after pivoting.** When you change an approach, remove the old scaffolding.
3. **Know your platform.** LIFF apps should use `liff.line.me` URLs for anything shared inside LINE.
